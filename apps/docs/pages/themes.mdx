## Custom Components

You can override the default components for every data type by passing a `components` prop to the `Form` component.

```tsx
import { Form, IStringDefaultProps, INumberDefaultProps } from '@zodform/core';
import { z } from 'zod';

function TextField(props: IStringDefaultProps) {
  // ...
}

function NumberField(props: INumberDefaultProps) {
  // ...
}

const schema = z.object({
  name: z.string(),
  age: z.number()
});

const components = {
  string: TextField,
  number: NumberField
} as const;

export function MyForm() {
  return <Form schema={schema} components={components} />;
}
```

<Callout type="info">Note that we define the `components` object outside of any React component.</Callout>

## Mantine

The `@zodform/mantine` package provides components built using [Mantine](https://mantine.dev/).

### Installation

We need to install the `@zodform/mantine` package and its peer dependencies.

```bash
pnpm i @zodform/mantine @mantine/core @mantine/hooks @babel/core
```

### Usage

```tsx
import { Form } from '@zodform/core';
import { components } from '@zodform/mantine';
import { z } from 'zod';

const schema = z.object({
  name: z.string(),
  age: z.number()
});

export function MyForm() {
  return <Form schema={schema} components={components} />;
}
```

The mantine theme is just a `components` object, much like the one
we defined in the [Custom Components](#custom-components) section.
